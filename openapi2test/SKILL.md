# openapi2test - OpenAPI 规范转 pytest 测试脚本

## 快速开始

### 输入格式
- **OpenAPI YAML/JSON 文件**：支持 `.yaml`、`.yml`、`.json` 格式
- **URL**：支持直接传入在线 OpenAPI 文档 URL
- **粘贴内容**：直接粘贴 OpenAPI 规范文本

### 输出格式
生成 Python pytest 测试脚本，每个接口对应一个测试函数：
- 文件名：`test_{api_name}.py`
- 测试函数命名：`test_{method}_{path_slug}`
- 包含请求发送、参数校验、响应断言

---

## 转换规则

### 1. 解析 OpenAPI 规范

解析以下关键字段：

| 字段 | 说明 |
|------|------|
| `openapi` | 版本号（仅支持 3.0+） |
| `info.title` | API 名称 |
| `servers` | Base URL |
| `paths` | 接口路径和方法 |
| `components.schemas` | 数据模型定义 |
| `components.securitySchemes` | 认证方式 |

### 2. 生成测试函数

每个接口路径 + HTTP 方法生成一个测试函数：

```python
def test_{method}_{path_slug}(base_url):
    """Test {method} {path}"""
    # 1. 构造请求
    # 2. 发送请求
    # 3. 断言响应
```

### 3. 认证处理

根据 `securitySchemes` 自动处理认证：

| 认证类型 | 处理方式 |
|---------|---------|
| `apiKey` | 自动添加到 header 或 query |
| `oauth2` | 生成 token（使用 mock 或环境变量） |
| `http` (Bearer) | 自动添加 Authorization header |

### 4. 参数校验规则

根据参数位置（`in`）校验：

| 参数位置 | 校验内容 |
|---------|---------|
| `path` | 必填、类型匹配 |
| `query` | 必填、枚举值、默认值 |
| `header` | 类型匹配 |
| `requestBody` | JSON Schema 校验 |

### 5. 响应断言

自动生成断言：

```python
# 状态码断言
assert response.status_code == 200

# 响应体结构断言（基于 schema）
assert "id" in response.json()
assert response.json()["name"] == "expected"

# 响应时间断言
assert response.elapsed.total_seconds() < 5
```

## 输出示例

对于 PetStore OpenAPI 规范，生成的测试脚本：

```python
"""
Generated by openapi2test
API: Swagger Petstore - OpenAPI 3.0

import pytest
import requests
from typing import Dict, Any


BASE_URL = "https://petstore3.swagger.io/api/v3"


class TestPet:
    """Pet 相关的接口测试"""

    def test_get_pet_by_id(self, base_url):
        """Test GET /pet/{petId} - Find pet by ID"""
        response = requests.get(f"{base_url}/pet/10")
        assert response.status_code == 200
        data = response.json()
        assert "id" in data
        assert "name" in data

    def test_add_pet(self, base_url):
        """Test POST /pet - Add a new pet to the store"""
        payload = {
            "name": "doggie",
            "photoUrls": ["https://example.com/dog.jpg"],
            "category": {"id": 1, "name": "Dogs"},
            "status": "available"
        }
        response = requests.post(f"{base_url}/pet", json=payload)
        assert response.status_code == 200
        data = response.json()
        assert data["name"] == "doggie"

    def test_update_pet(self, base_url):
        """Test PUT /pet - Update an existing pet"""
        payload = {
            "id": 10,
            "name": "updated_doggie",
            "photoUrls": ["https://example.com/updated.jpg"],
            "status": "available"
        }
        response = requests.put(f"{base_url}/pet", json=payload)
        assert response.status_code == 200

    def test_find_pets_by_status(self, base_url):
        """Test GET /pet/findByStatus - Finds Pets by status"""
        params = {"status": "available"}
        response = requests.get(f"{base_url}/pet/findByStatus", params=params)
        assert response.status_code == 200
        assert isinstance(response.json(), list)

    def test_delete_pet(self, base_url):
        """Test DELETE /pet/{petId} - Deletes a pet"""
        response = requests.delete(f"{base_url}/pet/10")
        assert response.status_code == 200
```

## 处理流程

1. **解析规范**：读取 OpenAPI YAML/JSON，提取接口信息
2. **生成结构**：按 tag（pet/store/user）分组生成测试类
3. **生成函数**：为每个接口生成测试函数
4. **添加认证**：根据 security schemes 添加认证头
5. **生成断言**：基于响应 schema 生成基础断言
6. **输出脚本**：生成完整的 pytest 脚本

## 注意事项

- 生成的测试脚本需要配置实际的 `BASE_URL`
- 认证相关的 API Key 需要替换为实际值
- 部分接口可能有依赖关系（如需先创建再查询），可根据实际情况调整
- 对于复杂的数据模型，生成的断言可能需要根据业务逻辑调整
